Full-Stack GPT

Github Link: https://github.com/jkim574/fullstack-gpt

Full-Stack GPT is an innovative project that harnesses the capabilities of GPT (Generative Pre-trained Transformer) models in a full-stack environment. This project showcases a diverse range of applications, from document processing to interactive web applications, all powered by the advanced AI of GPT models. It demonstrates how GPT can be integrated into various aspects of a web-based platform, providing intelligent and automated solutions to complex tasks. Each file performs the following tasks:

  • DocumentGPT.py: This script creates an interface for a GPT-based chatbot that can answer questions about uploaded files (like .txt, .pdf, or .docx). It utilizes the langchain library for processing and embedding files, and streamlit for the web interface. Features include embedding file contents, managing chat history, and providing a chat interface where users can ask questions about their uploaded document.
  • PrivateGPT.py: Similar to 01_DocumentGPT.py, but focuses on privacy by processing data offline. It uses ollama embeddings for file embedding, suggesting a different language model or approach for handling private data. The script ensures that data is processed locally, without sending it to external servers.
  • QuizGPT.py: This file is designed to create quizzes from user-uploaded documents or Wikipedia articles. It uses langchain and streamlit to generate quizzes, where the GPT model formulates questions and answers based on the provided content. Users can upload files or search Wikipedia articles, and the script generates a quiz to test knowledge on the given topic.
  • SiteGPT.py: This script facilitate by asking questions about the content of a specific website. It uses a sitemap loader to process website content and a GPT model to answer user queries based on the website's content. The user interface allows for entering a website URL and posing questions, with the script processing and providing answers.
  • ChefGPT.py: ChefGPT.py is a key component of the project, leveraging the power of FastAPI and GPT technology to provide a unique culinary experience. This application utilize FastAPI for Efficient Web Service, and built on the modern and efficient FastAPI framework. FastAPI's asynchronous capabilities ensure that the application can handle multiple requests simultaneously. By inputting ingredients, ChefGPT utilizes advanced vector search technology to find and suggest authentic Korean recipes that include those ingredients. CheftGPT also integrates with GPT models and vector search technology. Using OpenAIEmbeddings from the langchain library, the application converts recipe data into vectors. This allows for efficient similarity searching, ensuring that the recipes returned are relevant to the user's input. The application features a straightforward HTTP GET endpoint (/recipes), which accepts ingredients as input and returns a list of matching Korean recipes. Each recipe is presented with the recipe details and preparation instructions, making it easy for users to follow along. ChefGPT also uses Cloudflare for creating temporary URLs, ensuring a secure and isolated environment for testing and experimentation.

Vision in Transformer

Github Link: https://github.com/jkim574/Simple-VIT

This project demonstrates the use of a pre-trained Vision Transformer (ViT) model for image classification, using the timm library. The project is divided into several steps:

  • Install the timm library.
  • Define the model name as "vit_base_patch16_224" and set the device to GPU or CPU.
  • Create a pre-trained ViT model using the timm library.
  • Transform the image for preprocessing. Here, the input image is divided into N patches (N=14x14 for ViT-Base) and transformed into a D=768 embedding vector by a trainable 2D convolution.
  • Download ImageNet labels and a test sample image (hammer.png).
  • Add a trainable positional embedding vector to each patch embedding vector to provide positional information. This enables the model to learn the distances between patches within the image and understand how similar the patches are to each other.
  • The N(=197) embedding vectors then pass through L(=12) encoders. The vectors are split into queries, keys, and values, which are each further split into H(=12) for multi-head attention.
  • Visualize the classification results to display the output probabilities for each class.

This project demonstrates the use of a pre-trained Vision Transformer model for image classification and visualizes intermediate outputs and components of the ViT model, such as patch embeddings, positional embeddings, and multi-head attention, to understand the complex relationships between the input patches.


Transformer and Multi-head Attention

Github Link: https://github.com/jkim574/Transformer

This project implements a custom Transformer architecture in PyTorch, which learns and predicts patterns in numerical sequences. The model employs custom positional encoding, an encoder-decoder Transformer block, and target masking for effective training. The Transformer model used in this implementation is composed of the following layers:

  • Positional Encoder: Adds positional information to the input embeddings to allow the model to encode sequence information. The custom PositionalEncoding class that calculates and applies positional encoding to the input sequence is included.
  • Embedding: Maps input tokens to continuous vector space.
  • Transformer Blocks: A structure consisting of self-attention layers and feedforward layers stacked in sequence, allowing the model to understand relationships between input tokens.
  • Output Layer: Maps the output of the Transformer blocks to the target vocabulary.

The model is trained using stochastic gradient descent (SGD) optimizer and the CrossEntropyLoss loss function, and includes a validation loop to evaluate the model's performance after each epoch. It is trained for 10 epochs, and evaluated by using the predict function to pass a sequence of numbers and predict the next expected number pattern in the sequence.
